文件夹及文件管理字段设计

字段名 描述 类型
id 主键 int
name 文件夹或文件名称，添加的时候需要校验是否已存在，同一级的名称不能重复 str
flag 类型，文件夹还是文件 int
pid 父节点id，第一级的文件夹为0，文件没有一级的，都是在文件夹中存放 int
filepath 文件路径,文件夹没有路径，只需要文件夹名称在前端显示即可，配合type字段使用 str
filetype 文件类型（图片，doc等） str
user_id 用户外键 int
create_time 创建时间 datetime

模型设计

分享表：shares

名称 说明 类型
id 主键 int
user_id 用户外键 int
share_url 分享的连接 str
share_pwd 分享的访问密码，四位，可以自己设置，也可以系统生成 sre
share_type 分享类型，是永久的，还是指定时间过期，有永久，一个月，七天，一天 int
start_time 如果不是永久类型，需要给开始时间和结束时间 datetime
end_time 如果不是永久类型，需要给开始时间和结束时间 datetime
state 开启关闭分享 int
nums 访问人数限制 int
create_time 分享连接创建时间 datetime

分享子表:share_items

名称 说明 类型
id 主键 int
share_id 分享主表的外键，一对多 int
doc_id 分享的文件夹或文件的外键，一条分享对应多个文件夹或文件 int

下载逻辑

  • 文件：直接下载
  • 文件夹
    • 不给文件夹下载按钮，只让点击进入下载文件
    • 点击文件夹下载传文件夹id，然后把该id下的所有文件下载下来
    • 把文件夹打包，里面所有文件及文件夹，如果文件夹层级很多会影响后端性能
