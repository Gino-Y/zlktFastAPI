Pydantic模型

Pydantic 是一个用来用来执行数据校验的 Python 库，限制数据类型
文档地址：

  • https://pydantic-docs.helpmanual.io/
  • https://blog.csdn.net/swinfans/article/details/89629641

一、使用

1.安装包

pip install pydantic

2.定义

from pydantic import BaseModel

class User(BaseModel):

3.调用

直接传值
    user = User(id=1,name="hallen")

字典的方式
    user = {"id":1,"name":"hallen"}

    u = User(**user)

二、模型属性

1.Optional：可选字段

当一个字段不是必须的，可以使用Optional，这样不传参的时候默认使用None
from typing import Optional
from pydantic import BaseModel

class User(BaseModel):
    id: int
    age: Optional[int]

2.默认值设置

如果不想使用None作为默认值，就不使用Optional，可以自己设置默认值

gender: str = "male"

3.Union：多种数据类型

from typing import Union

Union[int, str]   # 用or连接


或者：
    from typing import List

    List[int or str]

三、模型嵌套：

from enum import Enum

class Gender(str, Enum):
    man = "man"
    women = "women"

class Person(BaseModel):
    name : str
    gender : Gender

class Class(BaseModel):
    name : str
    teacher: Person

四、错误处理

1.常见触发错误的情况

  • 如果传入的字段多了会自动过滤
  • 如果传入的少了会报错，必填字段
  • 如果传入的字段名称对不上也会报错
  • 如果传入的类型不对会自动转换，如果不能转换则会报错

2.错误的触发

当不满足类型的时候，会触发ValidationError

访问错误：

  • e.errors()：返回输入数据中发现的错误的列表。

  • e.json()：返回一个表示errors的 JSON。 # 建议使用

  • str(e)：返回人类可读的错误表示。

示例：

from pydantic import BaseModel,ValidationError

class Person(BaseModel):
    id: int
    name: str



if __name__ == '__main__':
    try:
        p = Person(id="ss",name="hallen")    # id是个int类型，如果不是int或者不能转换int会报错
        print(p.json())
    except ValidationError as e:
        print(e.errors())

3.自定义错误信息

from pydantic import BaseModel,ValidationError,validator

class Person(BaseModel):
    name: str

    @validator("name")
    def aa(cls,v):  # 第一个参数必须是cls，第二参数表示实例化的值
        print(v)
        if v != "hallen":
            raise ValueError("不是hallen")
        # 这里必须return，不然值就不是实例化传的值了
        -=-,return v


if __name__ == '__main__':
    try:
        p = Person(name="hallen")
        print(p.json())
    except ValidationError as e:
        print(e.errors())

4.定义错误模板

也可以定义错误模板：

from pydantic import BaseModel,ValidationError,validator,PydanticValueError

class NotHallenError(PydanticValueError):
    code = 'not_hallen'
    # 这里的wrong_value参数和下面的参数名一致，名称随便起
    msg_template = 'value is not "hallen", got "{wrong_value}"' 


class Person(BaseModel):
    name: str

    @validator("name")
    def aa(cls,v):  # 第一个参数必须是cls，第二参数表示实例化的值
        print(v)
        if v != "hallen":
            raise NotHallenError(wrong_value=v)   
        # 这里必须return，不然值就不是实例化传的值了
        return v


if __name__ == '__main__':
    try:
        p = Person(name="hallen1")
        print(p.json())
    except ValidationError as e:
        print(e.errors())
