Async

如果你不清楚你函数里的调用是否异步，那就定义为普通函数

用于耗时操作，比如大文件的读写等，不用等待读写完成才能执行其他的操作

异步通常描述为并发（并发，并行，串行的区别）

  • 并发的关键是你有处理多个任务的能力，不一定要同时

  • 并行的关键是你有同时处理多个任务的能力

官方说：

  • 无论你是否使用async，FastAPI都会采用异步的方式处理。
  • 但是，如果你定义了async函数，函数体却是同步的调用（例：/a接口），将导致函数执行过程变成串行

参考文档：https://www.pythonf.cn/read/155594

当程序碰到关键字await test2()后，异步程序挂起后去执行另一个异步test2程序

当挂起条件消失后，不管test2是否执行完，要马上从test2程序中跳出来

一、async示例：

'''
@IDE ：PyCharm
@Author ：知了-海龙
@Date ：2021/6/25 11:16
@Email：hallen200806@163.com
@Desc ：
'''

import asyncio
import time

async def test1(i):
    print("========test1")
    r = await test2(i)
    print('------------')
    print(i)

async def test2(i):
    print("========test2")
    print(i)
    await asyncio.sleep(4)

# 创建一个事件循环
loop = asyncio.get_event_loop()
tasks = []
for i in range(3):
    task = asyncio.ensure_future(test1(i))
    tasks.append(task)


starttime = time.time()
# 监测task运行的情况，并返回结果
loop.run_until_complete(asyncio.wait(tasks))
endtime = time.time()
print("==========")
print(endtime-starttime)
loop.close()

如果await后面跟的test2函数不是异步函数，那么操作就只能等test2执行完再返回，无法在test2执行的过程中返回。

如果要在test2执行完才返回，也就不需要用await关键字了，直接调用test2函数就行。所以这就需要await后面跟的是异步函数了

@app.post("/send-email-phone/")
async def send_email(email: str,phone:str, background_tasks: BackgroundTasks):
    background_tasks.add_task(task_send_email, email)
    background_tasks.add_task(task_send_code, phone)   // 可以添加多个任务
    return {"message": "发送邮件后台发送中。。。"}
