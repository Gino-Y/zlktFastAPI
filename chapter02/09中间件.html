中间件

一个函数，在每个request处理之前被调用，且在每个response返回之前被调用

类似flask中的钩子函数

一、执行流程

1、首先接收访问过来的request。

2、然后针对request或其他功能执行自定义逻辑。

3、传递request给应用程序继续处理。

4、接收应用所产生的response。

5、然后针对response或其他功能执行自定义逻辑。

6、返回response

二、自定义中间件

1.使用

函数的顶部使用装饰器 @app.middleware("http")

例如：

    from fastapi import FastAPI, Request

    @app.middleware("http")
    async def add_process_time_header(request: Request, call_next):
        pass

这里必须使用async和await，call_next需要

2.函数参数说明

  • 接收Request参数
  • 一个函数 call_next 它将接收 request 作为参数
    • 这个函数将 request 传递给相应的 路径操作
    • 然后它将返回由相应的路径操作生成的 response
  • 做进一步的修改response再返回

3.示例：

@app.middleware("http")
async def set_cookie(request:Request,call_next):
    response = await call_next(request)
    response.headers["name"] = "hallen"
    response.set_cookie(key="username",value="zhiliao")
    return response

我们请求完成后，响应头里多了一个新增的请求头“name”

三、使用已有的中间件

app.add_middleware()

参数

  • 第一个参数是中间件函数
  • 其他的配置信息

示例：

app.add_middleware(
    CORSMiddleware,
    allow_origins=origins,
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

四、常用中间件

  1. HTTPSRedirectMiddleware：强制发来的请求协议必须是https或者wss
  2. TrustedHostMiddleware：强制发来的请求必须在Header信息中设置了Host选项，为了避免HTTP Host Header攻击。
  3. GZipMiddleware：请求的头信息 Accept-Encoding 字段带有"gzip"时，GZipMiddleware负责完成相应的返回结果处理

五、Request

文档：https://www.starlette.io/requests/

  1. request.client.host：访问的IP
  2. request.client.port：获取端口号
  3. request.method：请求方式
  4. request.url：获取请求的url
  5. request.headers["Accept"]：获取请求头的指定key
  6. request.query_params['search']：获取查询参数
  7. request.path_params['username']：获取路径参数
  8. request.cookies.get('username')：获取cookie
  9. request.json()：获取请求json
  10. request.body()：获取请求body
  11. request.app：获取应用程序
  12. request.state：获取有关请求的其他信息
