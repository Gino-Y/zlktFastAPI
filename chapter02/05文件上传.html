知了fastapi从入门到项目实战

文件上传

文件上传需要使用form-data的方式

一、单文件上传

1.安装包：和post里面的form-data安装的包一样

pip install python-multipart

2.导入包File和UploadFile

from fastapi import FastAPI, File, UploadFile

3.使用

1.接收文件，并以bytes类型保存在内存中，适合小文件
    @app.post("/files")
    def create_file(file: bytes = File(...)):
        with open("../upload/aa.jpg", "wb") as f:
            f.write(file)
        return {"file_size": len(file)}

2.上传文件到服务器
    # 单文件上传
    @app.post("/upload")
    async def upload(file:UploadFile= File(...)):
        print(file.file)
        rep = await file.read()
        with open("../upload/chapter02/"+file.filename,"wb") as f:
            f.write(rep)
        return "上传成功"

UploadFile比File的优点：
    1.存储在内存中的文件达到最大大小限制，超过此限制后，它将存储在磁盘中
    2.以很好地处理大文件，如图像、视频、大型二进制文件等，而不会消耗所有内存
    3.可以从上传的文件中获取元数据。

4.UploadFile

  • 具有的属性
filename: 是一个str带有上传的原始文件名（例如myimage.jpg）。
content_type: 是一个str与内容类型（MIME 类型/媒体类型）（例如image/jpeg）。
file: 文件对象
  • 具有的async方法
write(data): 写入data(str或bytes) 到文件。
read(size)：读取文件的size( int) 个字节/字符。
seek(offset): 转到文件中的字节位置offset( int)。
close(): 关闭文件


例如：
    await myfile.seek(0)       会转到文件的开头
    await myfile.read()
    注意：myfile是文件对象

二、多文件上传

和单文件上传的区别，需要申明为List类型，然后先循环

@app.post("/uploadfiles/")
# 多文件上传
@app.post("/uploads")
async def uploads(files:List[UploadFile] = File(...)):
    for file in files:
        rep = await file.read()
        with open("../upload/chapter02/" + file.filename, "wb") as f:
            f.write(rep)

    return "上传成功"
