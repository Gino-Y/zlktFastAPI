跨域：CORS

ip:端口号

http://127.0.0.1:8080

https://127.0.0.1:8081

跨源服务器请求，导致跨域有三种情况：

  • 协议（http，https）不一样
  • 域（baidu.com）不一样
  • 端口（80、8080）不一样

只要其中有一个不同就是跨域，比如前端和后端不在同一个服务器或者端口不一样或者协议不一样都会导致跨域

一般出现在前后端分离，前后端不分离就不会出现跨域，因为服务器端口都一样，在同一个进程里面

一、请求过程

在前后端分离的项目中，前端和后端如果部署在同一个服务，那么运行端口肯定不一样，

当前端发起请求到后端，这个时候发送的首先是option请求，而不是真正的请求，

后端拿到option请求后先判断有没有资格(权限)，如果没有就会报错，cors，如果有，则会继续请求你真正发起的请求

二、解决跨域的原理

后端给前端授权，让前端可以请求，需要指定要授权的服务，

比如前端是127.0.0.1:8080，那在后端就要授权这个服务，或者使用*允许所有的服务

三、解决跨域的实现步骤

使用CORSMiddleware 解决跨域请求问题

1.导入包

from fastapi.middleware.cors import CORSMiddleware

2.使用

app = FastAPI()

app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],          # 允许所有的服务跨域访问
    allow_credentials=True,
    allow_methods=["*"],          # 允许所有的方法，也可以指定post，get等
    allow_headers=["*"],
)




或者指定特定的服务可以跨域访问：
origins = [
    "https://baidu.com",
    "http://localhost:8080",
]

app.add_middleware(
    CORSMiddleware,
    allow_origins=origins,
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

三、add_middleware的参数详解

CORSMiddleware   指定中间件名称
allow_origins    允许的服务地址列表，协议+IP：端口号 / 协议+域名  / *代表所有
allow_origin_regex 使用正则匹配允许的服务地址列表
allow_methods    允许跨域的方法列表，默认为 ['GET']，*代表所有，可以指定POST，GET等，注意是大写
allow_headers    允许跨域的请求头列表，默认为 []
allow_credentials    指示跨域请求支持 cookies。默认是 False
expose_headers - 指示可以被浏览器访问的响应头。默认为 []。
max_age - 设定浏览器缓存 CORS 响应的最长时间，单位是秒。默认为 600。
