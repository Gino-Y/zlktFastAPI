知了fastapi从入门到项目实战

post请求

一、application/json

要求提交的数据是json格式

1.正常的类型申明方式

@app.post("/add/")
def add(article:dict):
    print(article)
    return article

2.申明Pydantic 模型方式，建议使用

  • 好处
    • 转换为相应的类型
    • 校验数据
    • 代码属性提示
    • 重复使用定义好的模型
    • 成为生成的 OpenAPI 模式的一部分，并且被自动化文档 UI 所使用
  • 使用:单一类型
from fastapi import FastAPI,Body



1.定义模型
    from pydantic import BaseModel
    class Article(BaseModel):
        id: int
        name: str
        count: int

2.申明类型
    @app.post("/add/")
    def add(article:Article):
      print(article.name)
      # 可以当成dict操作
      a = article.dict()
      print(a.get("id"))
      return article

请求参数,发送请求和上面的一样
{
    "id":111,
    "name":"xxx"
}


如果想拥有函数的参数作为key，类似下面这样：
{
    "article":{
        "id":111,
        "name":"xxx"
     }
}

后端可以这样设置：
@app.post("/add/")
    def add(article:Article= Body(..., embed=True)):    # 插入
      return article
  • 如果有多个类型参数，使用参数名称作为请求体中的键
class Article(BaseModel):
    id: int
    name: str

class User(BaseModel):
    id: int
    name: str

@app.post("/add")
def add(article:Article,user:User):
    return {"user":user,"article":article}

from fastapi import FastAPI,Body

count:int=Body(...)

举例：
@app.post("/add")
def add(article:Article,user:User,count:int=Body(...)):
    return {"user":user,"article":article,"count":count}

 "article":

  • 注意：

    • 多个类型参数使用时函数里面的参数名作为请求体的key
    • 单个类型参数使用时不需要函数参数作为key，直接定义属性即可

  • list,set,tuple等类型

    • 对列表等里面的元素没有类型约束：
    tags: list = []
对列表等里面的元素也有类型约束：
    tags: List[str] = []

set：
    tags: Set[str] = set()

  • 模型嵌套

    • 子模型：
    class Image(BaseModel):
        url: str
        name: str
模型：
    class Article(BaseModel):
        id: int
        name: str
        image: Optional[Image] = None

        }
    }

    • images: List[Image]

三、数据校验

除了pydantic自带的类型校验外，还想对字段的长度等进行校验，怎么做？

1.使用Field

from pydantic import BaseModel, Field

class Article(BaseModel):
    id: int = Field(..., gt=0, description="id大于0")
    name: str

Field 的工作方式和 Query、Path 和 Body 相同，包括它们的参数等等也完全相同

2.一些特殊的校验，比如url的校验

  • HttpUrl

    • from pydantic import BaseModel, HttpUrl

class Image(BaseModel):
    url: HttpUrl
    name: str
